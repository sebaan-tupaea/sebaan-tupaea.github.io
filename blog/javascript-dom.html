  3. Describe what the DOM is and an example of how you might interact with it:

    4. The Document Object Model is a way to represent and interact with the elements of a web page. It is like a map that shows the structure of a page, including headings, paragraphs, and images. For example, if you want to change the text in a paragraph, you can use JavaScript to find that paragraph by its ID and update the text. This lets you modify the content and layout of the page as needed.

  4. Explain the difference between accessing data from arrays and objects:

    5. Arrays: An array is like a list of items. You access elements by their position or index in the list, starting from 0. For example, if you have an array of favourite fruits, you might access the second fruit with fruits[1].

    Objects: Objects are like a collection of labelled boxes, where each label (key) is paired with a specific item (value). You access data by using the key. For instance, if you have an object representing a person with keys for name and age, you would access the name with person.name and the age with person.age.

  5. Explain what functions are and why they are helpful:

    6. Functions are like shortcuts in programming. They let you pack a bunch of instructions into one set of curly brackets, which you can use whenever you need. For example, you can create a function to calculate the total cost of items in your shopping cart. Functions keep your code neat, prevent you from repeating yourself, and make fixing issues easier.
